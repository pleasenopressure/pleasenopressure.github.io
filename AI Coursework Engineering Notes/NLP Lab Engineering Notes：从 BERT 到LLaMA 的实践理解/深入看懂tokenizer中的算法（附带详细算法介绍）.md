# 什么是tokenizer
一言蔽之，这就是个分词器。举一个我们生活中的应用场景就是每当你向chatgpt提问一句话，llm并不是直接把你的这句话直接进行分析推理的。首先你的输入要先拆分成一个一个的token（这个你可以理解为基本的信息单位metadata，类似于重量kg）。例如“I am an international student.”这句话在真正进入模型model之前会被拆分成\[I ，am ，an， international ，student]。当然你也可以从这个单词本身来理解tokenizer，意思就是用来token化的工具人。
# tokenize为什么重要
tokenize的重要性可以通过一个有趣的笑话理解：
从前有个书生，为人吝啬又傲慢。一天他去朋友家拜访，适逢下雨。朋友本不想留他吃饭，便写下一张纸条以此委婉表达，想让他走。
**原始句子：**

> **下雨天留客天留我不留**

**错误断句（朋友的初衷）：**

> **“下雨天，留客天，留我不？留！”**  
> （意思是：下雨天是留客人的天，我留你吗？那当然留啊！——这是一种讽刺，意思是书生自己觉得是客，实际上没打算好好款待。）

**书生的理解（重新断句）：**  
书生看着纸条，冷笑一声，重新断了句，觉得朋友是在热烈挽留他：

> **“下雨天，留客天。留我？不！留。”**  
> （意思变成：下雨天，真是留客人的日子啊！要留我吗？不用留了！[但我偏要留下来吃]。说完大摇大摆地坐下吃饭。）

这说明了不同的断句（分词方法）会产生很大的不同。
# tokenizer的发展
接下来介绍具体怎么分词。
## word -level（word2vec)
最开始分词以词为最小单位即token,分词主要靠空格（对多语言不友好，例如中文）。另外这个的维护成本巨高，随着时代发展不断有新的词被发明出来，非常自然的这种方法就遇到了OOV（out of vocabulary)问题,即如何处理未登录词。词的数量庞大是一方面问题，但另一方面是一种词的多种变型在这个word-level角度是完全不相干的独立词，这其实造成了很大的信息冗余。
## subword-level(BPE)
为了解决上述的问题，人们提出了subword的概念，其中Byte pair encodeing为一个典型。这个思想就是不再将单词作为token，而是将组成单词的更小部分作为token，这其实是有实际意义的，大家都在学习英语时或多或少听过词根词缀，这其实就是组成单词的一部分。总而言之通过subword的方式，解决了word-level中的OOV和信息冗余的问题。然而还有一个问题就是BPE这个方法（具体实现我后面会讲）依赖空格来对句子进行预分词（分解成单词）。而实际上类似于中文是没有空格的，即多语言支持差。
## SentencePiece
上述的问题其实聚焦在分词依赖空格实现，那么sentencePiece则是将特殊符号（例如空格）也作为普通字符处理，即不在需要预分词了。这使他天然支持多语言。
## Byte-level BPE
Byte‑level BPE（Byte‑Pair Encoding at the byte level）是一种特殊版本的 BPE，它把文本先转换成原始字节（0–255），再在字节序列上做 BPE 合并。它是 GPT‑2、GPT‑3、GPT‑4 系列使用的 tokenizer。它能处理任何语言、任何符号、任何乱码，不会 OOV。

# 重点SentencePiece，BPE(Byte-Pair Encoding)，Unigram
## 我先介绍传统的BPE算法：
原句为：" low lowest"
**算法流程** ：
    1. **预分词**：先通过空格将句子分成单词。["low", "lowest"]
    2. **初始化**：词汇表只包含所有单个字符。[`l`, `o`, `w`, `e`, `s`, `t`]
    3. **统计**：在语料库中统计所有相邻字符对的频率。
    4. **合并 (Merge)**：找到频率最高的字符对（如 'e' 和 'r'），将其合并为新符号 'er'，并加入词汇表。**注意这里合并只能在单词内部合并，字符不能跨单词合并，所以这里其实每个单词后面是会带有一个特殊的符号</w>来区分单词边界的**
    5. **迭代**：重复 k 次（k 是超参数），直到词汇表达到预定大小。
- **结果**：常见词（如 "the"）保留为完整单词；罕见词（如 "unhappiness"）被拆解为常见的子词片段。

## SentencePiece
SentencePiece 是 Google 开源的一个无监督文本分词和解分词（Unsupervised Text Tokenizer and Detokenizer）系统。它**不是**一个单一的算法，而是一个完整的工程工具包。

SentencePiece 内部原生支持两种主要的子词分词算法，你可以通过参数选择使用哪一个：

1. **BPE (Byte-Pair Encoding)**
    
2. **Unigram Language Model (一元语言模型)**
### BPE in sentencePiece
原句在这里视为：" low_lowest"
其中的BPE的算法步骤就是取消了预分词的过程，以及merge阶段没有单词边界的限制了。
### Unigram(算法涉及dp较复杂)
如果说BPE是合并的思想，那么Unigram就是筛选删除的思想。
Unigram是从一个极大的词表中不断删除不好的子词最终得到最优子词的过程。
其目标是找到一组子词，使得训练语料的整体概率最大（Maximum Likelihood）
它的流程是：
例如我现在有个句子是 internationalization

1. **先构造一个非常大的候选子词集合**（几十万甚至更多）

| 子词     | 概率   |
| ------ | ---- |
| i      | 0.1  |
| in     | 0.2  |
| inter  | 0.05 |
| nation | 0.01 |
| nat    | 0.03 |
| at     | 0.1  |
| t      | 0.05 |
| er     | 0.1  |
| al     | 0.2  |
| nal    | 0.05 |

2. 对每个句子，计算所有可能的子词切分方式的概率 （使用动态规划）
    
3. 评估每个子词对整体概率的贡献
    
4. **删除贡献最小的子词**
    
5. 重复步骤 2–4，直到词表缩小到目标大小（如 32k）

这就是为什么叫 **Unigram Language Model**： 它假设每个子词的出现是独立的（unigram）

这里动态规划表的构建我介绍一下（不感兴趣的可以跳过）
dp[i] = 从位置i到句子末尾的最小负对数概率（越小越好）
为什么用负对数？ 因为：
- 概率相乘 → 对数相加
- 最大概率 → 最小负对数
international
0123456789012
dp[13] = 0 # 空串概率为 1（log(1)=0）
dp[12] = -log(0.1)+dp[13]  # 这里假设单词‘l'的概率是0.1
以下的dp[11]可以由两种状态转移过来一个是’al'+dp[13],一个是'a'+dp[12]
dp[11] = -log(0.2)+dp[13]
dp[11] = -log(0.1)+dp[12] # 假设‘a’的概率是0.1
这里比较两个dp[11],取最小的一个就是从位置11到句子末尾，采取最好的子词划分方式达到整体概率最高的情况。
后面按照这个方式得到这样一个dp表
位置:   0    1    2    3    4    ...   10    11    12    13
字符:   i    n    t    e    r          n     a     l     <end>

dp:    5.3  4.1  3.2  2.1  1.5        0.7   0.3   0.1    0
best:  in   t    er   nat  ion        nal   al    l
现在我们就可以从头开始找最佳的子词划分方式了
best[0] = "in"      → 跳到位置 2
best[2] = "t"       → 跳到位置 3
best[3] = "er"      → 跳到位置 5
best[5] = "nat"     → 跳到位置 8
best[8] = "ion"     → 跳到位置 11
best[11] = "al"     → 跳到位置 13
最终划分为：in | t | er | nat | ion | al
这让 **指数级数量** 的切分方式的时间复杂度降到了 线性时间 **O(n × k)** 内找到最优切分（k 是平均可匹配子词数）。

### 顺便一提Unigram的优异点
由于这里dp[]事实上包含了多种不同的子词切分方式以及对应的概率。我们在训练时还可以做到这样一件事：
不是每次都用最优划分，而是按照概率随机取样划分方案：
例如最佳子词划分方案是：dp[0]+dp[2]+dp[3]+dp[5]+...dp[11]
但我也可以第一个随机取样了dp[5],即第一次划分是inter|
这就是防止模型过拟合，让模型更鲁棒、更泛化，即我们常说的regularization（正则化），关键是引入随机性。同样这里其实也可以从数据增强方面考虑，模型对同一句话采取了不同的划分方式然后输入，这就像是nlp问题中的数据增强（“随机旋转图片”），做过图像类训练数据增强的同学能听懂我在说什么。
